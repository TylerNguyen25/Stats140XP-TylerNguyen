Overview

Today we are going to build a simple web application with Flask. The app we construct will let us doing the following: 1. Allow the user to submit messages 2. Allow the user to view a sample of submitted messages

1. Enable Submissions

To start let us create a GitHub repository with our files. Here is the link to the one I created in this tutorial: https://github.com/TylerNguyen25/Flask-Webapp. Now, we create a submit html template so that we can do the following:

  • A text box for submitting a message
  • A text box for submitting the name of the use
  • A “submit” button

Let us start with a template called base such that we can extend our submit html template off it.

Next we create a new .py file named app.py which will store our three Python functions for database management: get_message_db(), insert_message(request), and delete_messages().

This is what our first function get_message_db() looks like:

from flask import Flask, render_template, request, redirect, url_for, abort, g
import sqlite3

app = Flask(__name__)

def get_message_db():
    try: #return database if it already exists
        return g.message_db
    except: #if it does not exist, create one
        g.message_db = sqlite3.connect("messages_db.sqlite")
        cmd = 'CREATE TABLE IF NOT EXISTS messages (message TEXT, handle TEXT)' 
        cursor = g.message_db.cursor()
        cursor.execute(cmd)
        return g.message_db

The function above returns our database if it already exists, and if it doesn’t we create one through the SQL command cmd.

The second funtion we write is insert_message(request), which will handle inserting a user’s message into the database.

def insert_message(request):
    db = get_message_db()
    conn = db.cursor()

    msg = request.form['message'] #store the input message from submit.html
    handle = request.form['name'] #store the input nameh handle from submit.html
    cmd = f"INSERT into messages VALUES ('{msg}','{handle}')" #insert these values into the database 
    conn.execute(cmd)
    db.commit()
    db.close()

This function inserts the message and handle to the database which the user submitted on the submit.html page.

And the final function we create for database management is delete_messages.

def delete_messages():
    db = get_message_db()
    conn = db.cursor()
    cmd = 'DELETE FROM messages' #delete all rows in database 
    conn.execute(cmd)
    db.commit()
    db.close()

This function deletes all messages from the database so that the user can verify that their message is in fact displayed in the database.

In addition to these functions, we write a function called main() to render_template() the submit.html template. When the instance is a GET method, we simply render submit.html. On the other hand, when the instance is POST, we call insert_message(request) to record the message to the database and then finally render submit.html. We also pass a message letting the user know that their message was received.

@app.route('/', methods = ['POST', 'GET'])
def main():
    if request.method == 'GET': #if the request is GET
        return render_template('submit.html') #render the submit.html file without doing anything
    else: #if the request is POST 
        msg = "Message received"
        insert_message(request)
        return render_template('submit.html', note = msg ) #render the submit.html file while submitting requests.

2. View Random Submissions

As of now, we have the ability to submit and record a message into a database. This section illustrates to users how to view these messages.

To do so, we write a function random_messages(n) which returns a collection of n random messages from our database.

def random_messages(n):
    db = get_message_db()
    conn = db.cursor()
    cmd = f'SELECT * FROM messages ORDER BY RANDOM() LIMIT {n}' #select n random messages from our database 
    messages = conn.execute(cmd).fetchall()
    db.commit()
    db.close()
    return messages

In order to view these random message, we must create a ask.html file.

{% extends "base.html" %}

{% block header %}
<h2>{% block title %}View Messages{% endblock %}</h2>
{% endblock %}

{% block content %}
<ul>
    {% for message in messages %} #for each message out of our n random messages
        <p>{{message[0]}}</p> #display the message 
        <p>- <em>{{message[1]}}</em></p> #display the name handle 
        <br>
    {% endfor %}
</ul>
{% endblock %}

In this html file, messages is a list of tuples. Hence in order to access the message and user handle, we must construct a for loop and then index each indiviual message tuple. Here, message[0] contains the message and message[1] contains the user handle.

Finally, we create ask to render ask.html.

@app.route('/view')
def ask():
    messages = random_messages(5)
    return render_template('ask.html', messages = messages) #render ask.html with 5 random messages

In ask() we called random_messages(5) to generate five random messages from the database and then pass it into the render_template as an argument. This is the list of tuples which the ask.html file iterates over.

The last function in our file is the deleted() function. This function calls delete_messages() to clean all messages in our databse and returns a message to the user after doing so.

@app.route('/delete')
def deleted():
    delete_messages()
    msg = "Database has been cleaned"
    return render_template('delete.html', note = msg)
